Lien vers la doc officielle:
    https://reacttraining.com/react-router/web/guides/quick-start

Installation de la librairie de routing:
    Dans le terminal, en étant dans le dossier racine du projet, tapper:
        npm install --save react-router-dom
        
        
Etapes de mise en place du routing:
    a- Les imports qu'il faut:
        
        import { BrowserRouter as Router, Switch, Route, Link } from 'react-router-dom';
        
        
    b- Envelopper notre application par le router
    
    c- On met le Switch qui va gérer l'affichage conditionnel
    
    d- On définit les routes à l'intérieur du Switch, le Switch s'occupera de l'affichage conditionnel 
    
    e- On met en place nos Link ou NavLink afin de permettre à l'utilisateur de pouvoir naviguer entre nos pages.
    
    Description des composants indispensables:
    
    <BrowserRouter> : C'est un router qui utiliser l'API hystory d'HTML5, en l'utilisera comme parent global de notre app.
    
    <Switch>: s'occupe de l'affichage conditionnel, il rend le premier enfant dont le path et compatible avec la route présente sur la bare d'adresse du navigateur. Switch rendra (fera le rendu) qu'une seule Route seulement, la première qui match
    
    <Route>:  Le composant le plus important àbien comprendre dans React Router. Son rôle le plus basique est de faire le rendu d'une partie d'UI (truc qui se voit à l'écran) lorsque son path match l'URL courrante.
    
    <Link> et <NavLink>: Fournissent une navigation déclarative pour et dans notre application.
    
    Rq: Switch, Route et Link/NavLink, devront impérativement être en dessous du Router dans la hiérarchie des composants (fils, ou fils de fils).
        
    Exercice:
        Transformer la gestion de navigation des pages "Compteur" et "Contenu mappé" de la solution utilisant le state vers une gestion par react-router-dom en spécifiant les path adéquats.
    
    Composants d'ordre supérieur ou HOCs:
        Ce sont des composant qui utilisent dans leur code explicitement les children. Ils sont communiément utilisés pour injecter des fonctionalité implémentée en externe (librairie externe par exemple). Ils permettent une modularité fonctionnelle.
        
        useParams: hook permettant d'accéderauxparamètres de la route à partir de n'importe quel composant 
        useHistory: hook permettant de provoquer un changement dans l'history HTML à partir de n'importe quel composant
        *n'importe quel composant: composant react, mis sous un Router tout de même!
